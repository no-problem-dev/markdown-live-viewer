# Role（役割）

Git ブランチ管理担当として振る舞え。ブランチ命名規則と運用ルールに従い、適切なブランチを作成する。

---

# Request（依頼）

現在のコンテキストに基づき、適切なブランチを作成せよ。
また、ユーザーの判断は不要

## 実行内容

1. **現状確認**
   - `git status` で現在のブランチと状態を確認
   - 未コミットの変更がある場合は警告

2. **ブランチ種別の決定**
   - ユーザーの入力または作業内容から種別を判断

3. **ブランチ名の生成**
   - 命名規則に従い、簡潔で意味のあるブランチ名を生成

4. **ブランチ作成**
   - `develop` から新規ブランチを作成

---

# Route（実行順序）

```
1. git status で現状確認
2. ブランチ種別・名前の決定
3. git checkout develop && git pull origin develop
4. git checkout -b <branch-name>
5. 作成完了の報告
6. 指示されているタスクに取り掛かる
```

---

# Rule（制約・禁止事項）

## ブランチ命名規則

| 種別             | プレフィックス | 用途                                           | 例                        |
| ---------------- | -------------- | ---------------------------------------------- | ------------------------- |
| 機能開発         | `feat/`        | 新機能の追加                                   | `feat/add-login-page`     |
| バグ修正         | `fix/`         | バグの修正                                     | `fix/auth-redirect-error` |
| ドキュメント     | `doc/`         | ドキュメント(仕様やプロンプトを含む)のみの変更 | `doc/update-readme`       |
| リファクタリング | `ref/`         | 機能変更を伴わないコード改善                   | `ref/extract-auth-hook`   |

## 命名ガイドライン

- **英語**で記述（日本語禁止）
- **ケバブケース**（`kebab-case`）を使用
- **簡潔**かつ**意味がわかる**名前にする
- 長すぎる名前は避ける（目安: 30文字以内）

## 禁止事項

- `main` ブランチへの直接コミット
- `develop` への直接コミット
- 種別プレフィックスなしのブランチ名
- 日本語(ローマ字含む)・スペース・特殊文字を含むブランチ名

## ブランチ構造

```
main          ← 本番環境
  └── develop ← 開発環境
        └── feat/* ← 機能開発
        └── fix/*  ← バグ修正
        └── doc/*  ← ドキュメント
        └── ref/*  ← リファクタリング
```

---

# 入力例

```
# 種別を明示する場合
/gh01-branch feat/user-profile

# 説明から判断させる場合
/gh01-branch ログイン機能の追加

# 種別のみ指定
/gh01-branch fix
```
